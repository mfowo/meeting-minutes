# meeting-minutes プロジェクト

## 概要
慶應義塾体育会馬術部の会議（Zoom）の文字起こしVTTファイルから議事録MDを自動生成するツール。

## 使い方
```bash
python main.py <VTTファイルパス> [オプション]

# 例
python main.py ~/Downloads/meeting.vtt --context "慶應馬術部 運営ミーティング"
```

出力は `output/` フォルダに保存される。

## プロジェクト構成
- `main.py` — エントリーポイント
- `src/` — 処理モジュール群
- `config/glossary.csv` — 用語集（人名・馬名・専門用語）
- `output/` — 生成された議事録MDの保存先
- `KNOWLEDGE.md` — ノウハウ・フィードバックの蓄積

---

## Claude Code との対話フロー

### 議事録レビュー時のルール

ユーザーが議事録の修正フィードバックを与えたとき、Claude Code は以下を**自動的に**行う：

1. **glossary.csv を更新**
   - 人名・馬名・地名の表記訂正 → 正式名称と別名・誤変換例を登録
   - 既存エントリがある場合は別名に追記

2. **KNOWLEDGE.md を更新**
   - 訂正パターンをカテゴリ別に追記（人名/馬名/馬術用語/表記ルール）
   - 日付と更新内容を更新履歴に記録

3. **改善提案をする（推奨）**
   - 今回のフィードバックから読み取れるパターンを提示
   - 例：「今回の修正から、カタカナ語の長音符（ー）の有無がよく誤変換されています。他に同様のリスクがある馬名はXXXです」
   - glossary.csv や KNOWLEDGE.md に追加すべき内容を提案

### セッション開始時のルール

議事録を生成・レビューするセッションでは、必ず以下を最初に読み込む：
- `KNOWLEDGE.md` — 過去のノウハウを参照
- `config/glossary.csv` — 正式表記を確認

### フィードバックの受け取り方

ユーザーからの修正は以下の形式で来ることが多い：
- `誤表記→正表記` の羅列（例：`ラビー→ラヴィ`）
- 文章での指摘（例：「途中棄権は失権が正しい」）

どちらの形式でも、上記の自動更新を実行する。
