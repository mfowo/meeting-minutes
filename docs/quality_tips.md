# 議事録品質向上ノウハウ

## 1. ネクストアクションの抽出精度を上げる

### 見落としやすいパターン

**確約・申し出の表現**
会話の流れで自然に決まるタスクは、明示的な「お願いします」がなくても発生する。

| 表現パターン | 例 |
|---|---|
| 「〜しておきます」「〜しますね」 | 「明日朝一にメールしておきます」 |
| 「〜してもらっていいですか」→「はい」 | 質問→承諾の対話形式 |
| 「〜ということでよろしいですか」→「はい」 | 確認→合意の流れ |
| 「〜お願いできますか」→「わかりました」 | 依頼→受諾 |
| 誰かが名指しで言われて「あ、はい」と答える | 「高島さん、これ更新しておいて」→「わかりました」 |

**日程・期限の言及**
- 「○日までに」「翌日」「今週中に」「速やかに」
- 「○時に」「朝一に」「終わり次第」

**副次的に決まるタスク**
メインの議題を決めた際に、それに付随して生まれる小さなタスクを見落としやすい。
例：「フレンドシップ廃止」→副次タスク「伊藤くんに連絡・了承を得る」

### プロンプトへの追記（Claudeに指示する場合）

```
ネクストアクションを抽出する際は以下に注意してください：
1. 「〜しておきます」「わかりました」など暗黙の承諾も含める
2. 決定事項から派生する連絡・確認タスクも漏らさない
3. 担当者が名指しされた瞬間を必ずタスクとして記録する
4. 期限の言及（日付・「速やかに」「翌日」等）をタスクに付記する
5. 議事の最後ではなく、会話の途中で決まったタスクも見落とさない
```

## 2. 人名の扱い

### VTTファイルと人名

**Teams/Zoom VTTの話者名タグは信頼できる**
`<v 久保 知之>` のような話者名タグはユーザー自身がTeams/Zoomプロフィールに設定した名前なので、
文字起こし本文（誤変換が多い）とは別に信頼できる。

**文字起こし本文中の人名は誤変換が多い**
- 同音異字：「仁野」→「新納」、「2野」など
- 読み誤り：呼称・あだ名が正式名称に変換されない場合がある

**対策：用語集に人名を登録しておく**
`config/glossary_converted.csv` に正式名称・よみ・別名を登録することで、
補正ステップで自動修正される。

### 社馬連関係者の確認方法

議事録を他のツール（Gemini等）で作った場合、そちらの人名出力と照合して用語集を更新する。
→ `config/glossary_converted.csv` に `人名_連盟` カテゴリで追記。

## 3. 議事録フォーマットのベストプラクティス

### ネクストアクション表のフォーマット

表形式で担当者・期限・タスクを一覧化するのが実用的。

```markdown
| 担当者 | 期限 | タスク |
|--------|------|--------|
| 新納   | 翌日朝一 | 4団体への契約書郵送依頼メール送付 |
```

**「速やかに」「大学役員確定後」など期限が曖昧な場合も明記する。**

### 「持ち越し課題」セクションを設ける

決定事項・アクションアイテムに含まれない、次回以降で議論・確認が必要な事項を別セクションに記録する。
例：「電子印鑑の導入検討」「年間行事予定の確認」

## 4. Claudeへの指示プロンプト改善案

`src/minutes_generator.py` のプロンプトに以下を追加すると精度が上がる。

```
## 議事録作成上の注意
- ネクストアクションは担当者・期限・タスク内容の3点セットで表形式にまとめる
- 決定事項から派生する連絡・確認タスクも漏らさない
- 未解決のまま持ち越された課題を「持ち越し課題」セクションに分けて記載する
- 会話の中で自然に承諾されたタスク（「あ、はい」「わかりました」）も拾う
- 人名は用語集の正式名称を優先して使用する
```

## 5. コスト最適化

| 状況 | 推奨 |
|------|------|
| 毎回・定期的な会議（同じメンバー） | スクリプト自動化（用語集が育つほど精度UP） |
| 月1〜2回・コストが気になる | Claude Codeチャットに貼り付けて手動処理 |
| 2時間超の会議 | チャンク分割処理が必要（スクリプトが有利） |

APIコスト目安（Claude Sonnet）：2時間会議1回あたり **$0.05〜$0.30**
